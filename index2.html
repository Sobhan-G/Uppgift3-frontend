Dokumentobjektmodellen

Dokumentobjektmodellen (Document Object Model, DOM) är ett programmeringsgränssnitt (Application Programming Interface, API) för bl.a. HTML och XML-dokument. Genom att DOM översätter dokumentet till noder och objekt kan programmeringsspråk kopplas till dokumentet och ändra dess struktur, stil och innehåll.
HTML DOM är en standard modell för programmering för HTML. Den definerar:



•HTML element som objekt
•Egenskaperna för alla HTML element
•Metoder att komma åt alla HTML element
•events för alla HTML element
HTML DOM är alltså en standard för hur man lägger till, tar bort, ändrar eller hämtar HTML element.


•JavaScript kan ändra alla html element på sidan
•JavaScript kan ändra alla HTML attribut på sidan
•JavaScript kan ändra alla CSS stilar på sidan
•JavaScript kan ta bort existerande Html element och attribut.
•JavaScript kan lägga till nya Html element och attribute.
•JavaScript kan integrera med alla existerande html element på sidan. (react to all existing HTML events in the page)
•JavaScript kan skapa nya html events på sidan.

Event Bubbling or Event Capturing?

Event-prioriteringar är ett sätt att prioritera elementen när ett event sker.

Det finns 2 olika sätt att göra Event-prioriteringar i HTML DOM Bubbling och Capturing
Om du har en (p) element i en (div) och användaren klickar på (p) elementet vilket element ska hanteras först!?


I bubbling så hanteras den inre elementet först sen den yttre : först (p) sedan (div)


I capturing så hanteras den yttre elementet först sen den inre : först (div) sedan (p).

Med addEventListener metoder kan du specificera prioriteringen genom att använda ”useCapture” parameter.
Default värdet är falsk, Vilket gör att den använder bubbling prioriteringen, När värdet är Sant används i stället Capture prioriteringen först.

Historik

DOM designades för att vara plattformsoberoende och rekommenderas sedan 1998 av W3C. Första versionen av DOM stöddes av Netscape 3 och Internet Explorer 3 (1996).

Så funkar DOM

När en webbsida läses in skapar DOM en modell av dokumenten där dess element, attribut och innehåll representeras som objekt. Modellen är strukturerad som ett träd men kan också beskrivas som en familj eftersom alla “grenar”, eller noder som de också kallas, relaterar till varandra utifrån släktskapstermer.

Utforska DOM-strukturen

Med hjälp av webbläsarens Inspektera-funktion kan användaren utforska webbsidans DOM-struktur. Markera ett särskilt element, högerklicka och välj “Inspektera”, och gå till Elements. Välj därefter Properties för att se vilka egenskaper som hör till varje document object / element.

Fler funktioner i Firefox och tilläggsprogram

Webbläsaren Firefox har flera funktioner för att utforska DOM-strukturen. Bl.a. kan man via Firefox få tillgång till en 3d-vy över hur trädet ser ut och hur de olika objekten relaterar till varandra. Författaren till JavaScript & jQuery rekommenderar även Firefoxtillägget Firebug.

Noder

Allt som finns i HTML-dokumentet är en nod. Det finns olika typer av noder. Hela dokumentet är en dokumentnod (document) och fungerar som utgångspunkt för alla andra noder. Varje html-element är en elementnod (element), t.ex. html, body, h1, p. Texten inuti ett html-element är en textnod. Varje html-attribut är en attributnod, ex. class, href, src, id. Alla kommentarer är kommentarsnoder.

Nodernas relationer

Noderna relaterar till varandra utifrån en familjehierarki och beskrivs med begreppen förälder, barn och syskon. En nod kan bara ha en förälder, utom root (html) som saknar förälder. En nod kan ha flera barn, där barnen är syskon till varandra.

H1 och p är syskon och barn till body, som är barn till html. H1:s respektive p:s barn är dess textinnehåll. Textinnehållet utgör alltså en egen nod och är inte en del av elementnoden.

Metoder och egenskaper

Objekten document och window är de vanligaste gränssnitten som används i DOM.

Nodtyperna ärver metoder ovanifrån i trädstrukturen och kan ha egna metoder som är anpassade för den typ av data som det särskilda objektet hanterar.

Elementnoderna

Olika nyckelord ger åtkomst till ett specifikt eller en grupp av element utifrån id, css-selektor eller html-tagg.

Även nyckelord för att nå specifik artikel i en HTMLCollection eller NodeList som är arrayliknande samlingar av element.

Det vanligaste sättet enligt W3Schools att nå ett html-element är att identifiera det med hjälp av dess id-värde. Det lättaste sättet att få tillgång till elementets innehåll är genom att använda egenskapen innerHTML. Egenskapen kan användas för att nå eller ändra valfritt html-element, inklusive html och body.


Finding HTML Elements

document.getElementById(id) Find an element by element id
document.getElementsByTagName(name) Find elements by tag name
document.getElementsByClassName(name) Find elements by class name

Attributnoder

De egenskaper som hör till html-element kan nås med specifika nyckelord och hämta, skriva eller ta bort egenskaper hos elementet.


Changing HTML Elements (Property)
element.innerHTML = new html content, Change the inner HTML of an element
element.attribute = new value, Change the attribute value of an HTML element
element.style.property = new style, Change the style of an HTML element

Change the attribute value of an HTML element
element.setAttribute(attribute, value)
Method Description - Adding and Deleting Elements
document.createElement(element) Create an HTML element
document.removeChild(element) Remove an HTML element
document.appendChild(element) Add an HTML element
document.replaceChild(new, old) Replace an HTML element
document.write(text) Write into the HTML output stream
